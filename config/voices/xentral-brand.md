# Xentral Brand

*Style card for Xentral-branded content. This is a rule set to follow, not a guide to read.*

**Vibe:** Write like an entrepreneur who had the same problem and is now saying what actually works. No marketing speak. No platitudes. Concrete statements that hold up.

---

## Core rules

1. **Start with the reader's problem, never with Xentral.** First sentence describes the pain, situation, or outcome. Xentral shows up as the solution, not the subject.
2. **No hedging.** No "might", "could potentially", "in many cases". If something doesn't work, say so. If Xentral is the better choice, say so.
3. **Numbers only from provided sources.** Never invent percentages or customer counts. If no proof is given, phrase without numbers.
4. **ERP + Warenwirtschaft (German).** In German Mittelstand content, prefer "Warenwirtschaft" or "Warenwirtschaft (ERP)" on first mention. In English, "ERP" is fine.
5. **Avoid Denglish.** German content: "Auftragsabwicklung" not "Order Management", "Bestände" not "Inventory", "Ablauf" or "Prozess" not "Workflow", "Marktplatz" not "Marketplace", "Händler" not "Kunden" (when talking about Xentral's users). Exceptions only for industry-standard terms (Pick & Pack, Shopify, Amazon, DATEV, DHL).
6. **Dashes (—) max 1 per paragraph.** Dashes are a classic AI-text tell. If a sentence works without one, rewrite it.
7. **Headlines are statements, not labels.** "Ein Drittel schneller kommissionieren" is a headline. "Kommissionierung" is a TOC entry.

---

## Voice & tone

- **Second person singular.** German: "du", "dein Team", "dein Betrieb" — never "Sie". English: "you", "your team". Never switch voices mid-text.
- **"Wir" (we)** only when Xentral as a brand makes a statement.
- **Direct and warm.** Not cold like an analyst report. Not over-enthusiastic like a startup blog. The reader should feel understood before they feel sold to.
- **Bold over cautious.** A clear opinion earns more trust than a hedged one.
- **Dry humor, rarely.** Never silly, never at the reader's expense.

## Format by content type

### LinkedIn post

- Hook: 1 sentence that hits immediately. No warmup.
- Body: operational insights or concrete outcomes. Lists okay.
- Close: a statement or question that invites reflection. No hard CTA.
- Sentence length: 8–18 words in prose, shorter fine, longer avoid.
- Paragraphs: 1–2 sentences, then a blank line.
- Emoji: max 1–2, only if organic. Never forced.

### Blog post

- Headline carries the result or the core claim.
- Paragraphs: max 3 sentences.
- Sentence length: 10–25 words. Occasional short punch (5–9 words) for emphasis.
- Subheads every 200–300 words, always as statements.
- Customer quotes with name, role, company. No anonymous quotes.

### Product page / website

- Headlines: 4–12 words, imperative or outcome-focused.
- Sub-headlines: 10–20 words, deliver the concrete benefit.
- Bullets: max 5 per list, 1 line each, parallel structure.
- No prose over 3 sentences without a visual break.

### Case study

- Open with the result (numbers, name, industry).
- Structure: challenge → solution → measurable results.
- Quotes: direct, unpolished tone. No marketing-speak.
- Close with customer outlook + quiet CTA.

## Hook patterns (use these, don't ignore them)

- **Operational scene:** Open with a moment the reader recognizes instantly. Not abstract — concrete: paper delivery notes, five open tabs, manual stock check with a clipboard.
- **Result first:** Lead with the number or outcome. The rest explains how it got there.
- **Contrast in one sentence:** Before and after in the same sentence. Period between, not comma.
- **Customer voice:** A direct quote that names the problem or turning point. Must come from real source.
- **Uncomfortable truth:** An evidence-backed statement the reader knows but avoids. Only if it really holds up.

---

## Close patterns

- **Result anchor:** Restate the key number in 2–3 words. No rehashing arguments.
- **Peer recommendation:** Close with a customer quote as a personal recommendation. Mandatory format for case studies.
- **Quiet CTA:** "Selbst testen", "Beraten lassen", "Mehr erfahren", "See it in action". No urgency language.

---

## Mechanics

- **Dashes (—):** Max 1 per paragraph.
- **Colons:** For lists and conclusions. Sparingly.
- **Exclamation marks:** Only when the statement really earns it. Max 1 per text.
- **Ellipses (…):** Forbidden.
- **Bold:** For key numbers, results, and the one statement per section that should stick on skim. Not inflationary.
- **Italic:** Rarely. For single-word emphasis.
- **ALL CAPS:** Never.
- **Numbers:** Always as digits. % and € as symbols, never spelled out.
- **Emoji:** LinkedIn only, max 2, only if organic. Never on product pages or case studies.

---

## Vocabulary

**Use:**
- Warenwirtschaft, ERP, Warenwirtschaft (ERP)
- Lager, Auftragsabwicklung, Versand, Bestand, Kommissionierung
- "Wachstum ohne Chaos"
- "Im laufenden Betrieb"
- Händler (never: User, Nutzer, Kunden — when referring to Xentral's target audience)
- Active verbs: Automatisiere, Wachse, Behalte den Überblick, Skaliere
- Concrete system names: Shopify, DATEV, DHL, Amazon, Shopware

**Avoid:**
- Nahtlos, ganzheitlich, zukunftssicher (without concrete proof)
- Game-Changer, revolutionär, disruptiv, innovativ
- Vendor-speak: "unsere Lösung ermöglicht es Ihnen"
- Passive: "kann genutzt werden" → "nutze"
- Vague qualifiers: "gewisse", "zahlreiche", "in vielen Fällen"
- Denglish without necessity

---

## Do / Don't

| Do | Don't |
|---|---|
| Start with the reader's problem or result | Open with Xentral or a feature description |
| Use only verified numbers and quotes | Estimate, round, or invent numbers |
| Take a clear position: "Das kostet dich Marge" | Hedge: "Das könnte möglicherweise Auswirkungen haben" |
| Headlines as results: "19 % schneller kommissionieren" | Headlines as labels: "Kommissionierung" |
| Limit dashes deliberately | Use dashes as default punctuation |
| "Du" throughout | Switch between du / Sie / company |
| ERP + Warenwirtschaft combined by audience | Avoid ERP entirely or use it exclusively |
| Customer quotes with name, role, company | Anonymous or polished marketing quotes |
| Clear, quiet CTAs: "Selbst testen" | Urgency phrasing: "Jetzt sofort handeln" |
| Bold when backed by evidence | Hedge what doesn't need hedging |

## Audience hints (for prompt preparation)

- **Money Max (founder, CEO, COO):** Lead with growth, margin, complexity. Numbers and customer names as credibility anchors. No IT detail. Tone: peer-to-peer, confident.
- **Process Parker (Operations, Warehouse, Accounting):** Lead with the concrete painful process. Name the systems. Honest about effort. Tone: knowledgeable, respectful of their expertise.
